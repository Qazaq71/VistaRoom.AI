# ACS-002 — Scene Intelligence

**Статус:** AI Capability (сервисный модуль)
**Соответствует:** Living Strategic Roadmap v1.3, Level 3 — Scene Intelligence
**Обслуживает:** PCS-001, PCS-002 (через ACS-001), PCS-003, PCS-004, PCS-005, PCS-007
**Версия документа:** 1.0

---

## Purpose

Единая точка понимания входных данных пользователя и содержимого помещения. Объединяет три компонента, указанных в Roadmap v1.3: Input Handler (нормализация запроса), Room Analyzer (анализ фото), Structured Scene (единый объект-результат). Это базовый модуль, от которого зависит почти каждая Platform Capability — без него не может корректно работать ни полная генерация (PCS-001), ни частичное редактирование (PCS-002, через `maskRegion` в контракте ACS-001).

## Responsibilities

- **Input Handler:** объединять фотографию, текст, стиль, цвета, референсы, режим работы в единый нормализованный запрос (Unified User Request).
- **Room Analyzer:** анализировать фотографию помещения — определять стены, окна, двери, мебель, перспективу, свободное пространство.
- **Structured Scene:** формировать единый JSON-объект сцены, который становится источником истины для всех последующих модулей (Domain Intelligence, Prompt Intelligence и др.).
- Для режимов частичного редактирования (`PARTIAL_EDIT`/`INPAINTING` из ACS-001): по текстовому описанию цели редактирования пользователя (например, "заменить диван") находить соответствующий объект на Structured Scene и формировать `maskRegion` — конкретную область изображения, подлежащую изменению.

## Non-responsibilities

Модуль **не отвечает** за:
- принятие решений о стиле, материалах, цвете (Domain Intelligence) — Scene Intelligence только описывает, что есть в помещении, не решает, что должно быть;
- формирование промпта для генерации (Prompt Intelligence);
- вызов провайдера генерации (Generation Intelligence, ACS-001);
- расстановку мебели или рекомендации по зонированию (Layout & Ergonomics Intelligence, используется в PCS-007) — Scene Intelligence определяет текущее свободное пространство, но не предлагает, как его использовать;
- решение о повторном анализе при неудачном результате — инициируется извне (аналогично ограничению, зафиксированному в ACS-001 для Generation Intelligence).

## Inputs

| Поле | Обязательность | Описание |
|---|---|---|
| `photo` | Обязательно | Исходная фотография помещения. |
| `userText` | Опционально | Текстовое описание запроса пользователя (стиль, пожелания, для edit-режимов — цель редактирования). |
| `stylePreference` | Опционально | Явно указанный пользователем стиль. |
| `colorPreference` | Опционально | Явно указанные цветовые предпочтения. |
| `references` | Опционально | Референсные изображения. |
| `mode` | Обязательно | Соответствует режиму из ACS-001: определяет, требуется ли только Structured Scene (для `FULL_GENERATION`) или дополнительно `maskRegion` (для `PARTIAL_EDIT`/`INPAINTING`). |

## Outputs

| Поле | Описание |
|---|---|
| `unifiedUserRequest` | Нормализованный объект запроса пользователя (результат Input Handler). |
| `structuredScene` | JSON-объект сцены: тип пространства, объекты, освещение, геометрия, ограничения (структура согласуется отдельно, ориентир — формат, ранее обсуждавшийся для этого объекта: `spaceType`, `objects`, `lighting`, `geometry`, `constraints`). |
| `maskRegion` | Только для edit-режимов — область изображения, соответствующая целевому объекту редактирования. Обязателен к передаче в Generation Intelligence согласно контракту ACS-001. |

## Dependencies

- **Восходящие:** отсутствуют — это один из самых базовых модулей, зависящий только от сырого пользовательского ввода.
- **Нисходящие (потребители):** Prompt Intelligence (использует Structured Scene для построения промпта), Generation Intelligence (ACS-001, получает `maskRegion` для edit-режимов), Domain Intelligence модули (Furniture, Layout, Material, Color, Style Intelligence — используют Structured Scene как контекст), Project Intelligence (PCS-003, агрегирует Structured Scene по нескольким помещениям), Design Planning (PCS-007, использует геометрию и свободное пространство).

**Запрещённые зависимости:** Scene Intelligence не имеет права обращаться к Domain Intelligence, Prompt Intelligence, Generation Intelligence или Production-коду — она находится в начале цепочки и не должна знать о том, что происходит с её результатом дальше.

## Consumers

PCS-001, PCS-002 (косвенно, через ACS-001), PCS-003, PCS-004 (с оговоркой ниже), PCS-005 (косвенно, через Furniture/Material Intelligence), PCS-007.

## Public Contract

```
normalizeInput(rawInput: {
  photo: ImageRef,
  userText?: string,
  stylePreference?: string,
  colorPreference?: string,
  references?: ImageRef[],
  mode: "FULL_GENERATION" | "PARTIAL_EDIT" | "INPAINTING"
}) => UnifiedUserRequest

analyzeRoom(photo: ImageRef) => StructuredScene

deriveMaskRegion(
  structuredScene: StructuredScene,
  targetDescriptor: string  // например, "диван"
) => RegionDescriptor | ErrorNotFound
```

`deriveMaskRegion` обязан возвращать явную ошибку `ErrorNotFound`, если целевой объект не идентифицирован на Structured Scene — не приблизительную догадку по расположению, а именно отказ, аналогично принципу "явный статус вместо домысленного результата", уже зафиксированному для Design Reasoning (PCS-008).

## Extension Points

- Расширение схемы Structured Scene новыми полями (например, специфичными для коммерческих пространств зонами — барная стойка, ресепшн) должно быть аддитивным (не ломать уже существующих потребителей схемы) и версионируемым.
- **Открытый вопрос, перенесённый из PCS-004:** способность Room Analyzer распознавать типовые коммерческие зоны не подтверждена как решённая задача. Это расширение, а не базовая функциональность модуля — до подтверждения его технической реализуемости Structured Scene для коммерческих пространств может быть ограничена базовой геометрией (стены/окна/двери) без специфичных зон.

## Knowledge Packs

Базовая функциональность (геометрия, определение объектов) не зависит от Knowledge Packs — это задача компьютерного зрения, а не применения предметных знаний. Расширение на распознавание коммерческих зон (см. выше) может потребовать отдельных, специфичных для Knowledge Pack профилей распознавания — не подтверждено.

## Tests

- **Контрактные тесты:** `deriveMaskRegion` возвращает `ErrorNotFound` для несуществующего объекта, а не приблизительный регион.
- **Boundary-тесты:** Scene Intelligence не импортирует Domain Intelligence, Prompt Intelligence, Generation Intelligence, Production-код.
- **Golden-тесты:** фиксированный набор тестовых фотографий даёт стабильно корректное определение геометрии (окна, двери, свободное пространство) — точный критерий "корректности" требует отдельного определения (например, сравнение с ручной разметкой).
- **Регрессионные тесты** на схему Structured Scene — добавление новых полей не должно ломать потребителей, ожидающих старую версию схемы.

## Performance Expectations

Отдельных подтверждённых данных о стоимости и задержке именно для Room Analyzer (анализ фото, отдельный от вызова генерации изображения) в этом диалоге не предоставлялось. Ранее подтверждённые данные ACS-001 (≤ $0.061, ~60 сек) относятся к вызову Generation Intelligence, а не к этому модулю — если Room Analyzer использует отдельный vision-вызов (например, к LLM с поддержкой изображений), это дополнительная стоимость и задержка сверх уже посчитанной, которая нигде в этом диалоге не измерялась.

**Не подтверждено, требует отдельного замера** до фиксации бюджета этого модуля.
