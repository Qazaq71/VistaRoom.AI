# PCS-010 — Collaboration Platform

**Статус:** Platform Capability
**Соответствует:** Living Strategic Roadmap v1.3, Level 2, Capability 10
**Версия документа:** 1.0

---

## Purpose

Обеспечить совместную работу над проектом: дизайнер, клиент, семья, команда. Последняя capability в списке Level 2 — по формулировке из более ранних версий Roadmap, обозначена как направление "в будущем", то есть не первоочередная для реализации.

## Business Value

- Открывает Enterprise-сегмент (упомянутый в Decision Governance Roadmap v1.3 как отдельный KPI-профиль — "проекты команд", "совместная работа") — команды дизайнеров, агентства, работающие с несколькими клиентами.
- Позволяет монетизацию через Teams/API (Phase M — Production Platform), а не только через индивидуальные подписки.
- Расширяет применимость Project Intelligence (PCS-003) на сценарий, где над одним проектом работает не один человек, а несколько ролей с разными правами (дизайнер предлагает — клиент утверждает).

## User Stories

- Как дизайнер, я хочу поделиться проектом с клиентом для согласования до финальной реализации.
- Как клиент, я хочу оставить комментарии к конкретным элементам проекта (не всему проекту целиком), чтобы дизайнер понимал, что именно нужно изменить.
- Как семья, планирующая ремонт вместе, мы хотим совместно вносить правки в один и тот же проект.
- Как команда (агентство), мы хотим управлять несколькими проектами разных клиентов в рамках одного корпоративного аккаунта.

## AI Modules

Collaboration Platform не вводит собственных AI-модулей уровня Level 3 — это преимущественно инфраструктурная и продуктовая capability (права доступа, комментарии, совместное редактирование), использующая уже существующие AI-возможности (Project Intelligence, Personal AI Designer) в многопользовательском контексте.

## Knowledge Packs

Не применимо — эта capability не использует Domain Knowledge, а работает с правами доступа и совместным редактированием.

## Dependencies

- **Восходящие:** Project Intelligence (PCS-003) — совместная работа возможна только над уже существующей сущностью "проект"; Personal AI Designer (PCS-009) — у каждого участника совместной работы свой профиль предпочтений.
- **Нисходящие:** Production Platform (Phase M) — Teams и API как инфраструктурные элементы, поддерживающие эту capability на уровне биллинга и доступа.
- **Внешние:** отсутствуют, определённые в рамках этого диалога.

## Integration Gates

Отдельного явного Gate для Collaboration Platform в Level 4 Engineering Roadmap v1.3 не выделено. Это четвёртый подобный случай в серии PCS-006–PCS-010 (после Renovation Intelligence, Design Planning, Design Reasoning) — суммарно пять из десяти Platform Capabilities (PCS-006, 007, 008, 009 частично, 010) не имеют однозначного соответствия отдельному Gate в Level 4.

**Решение зафиксировано:** Roadmap v1.3 не изменяется. Соответствие Capability ↔ Integration Gate для всех пяти отмеченных случаев уточняется в рамках Decision Governance по мере реализации соответствующих фаз — это архитектурные зависимости, наблюдение зафиксировано и не требует немедленной правки документа.

## Performance Budget

Не применимо в терминах AI-вызовов — данная capability является инфраструктурной (права доступа, real-time синхронизация изменений между участниками), а не генеративной. Требует отдельной оценки затрат на инфраструктуру совместного редактирования (например, WebSocket-соединения для real-time синхронизации), что выходит за рамки шаблона Performance Budget, ориентированного на AI-вызовы.

## Success Criteria

Не определены в этом документе содержательно, так как сама capability обозначена как перспективная ("в будущем") в более ранних версиях обсуждения — фиксация точных критериев успеха для функциональности, которая не входит в ближайшие фазы реализации, была бы преждевременной.

## KPIs

Согласно Enterprise Profile из Living Strategic Roadmap v1.3: проекты команд, совместная работа, использование API, время подготовки проекта — уже определены на уровне самой Roadmap.

## Future Evolution

- Ролевая модель доступа (владелец/редактор/наблюдатель) — базовое требование для любой совместной работы, не детализировано в этом документе.
- Real-time уведомления об изменениях проекта другими участниками — отдельная техническая задача.
