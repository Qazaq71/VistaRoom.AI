## Рекомендуемая последовательность закрытия перед Phase C1

Это не жёсткое предписание, а порядок, логически вытекающий из зависимостей, уже установленных в самих документах:

1. **ADR-010** (Formatter/decisionTrace) и **ADR-006** (режимы Generation Intelligence) — оба напрямую блокируют Gate 1, самый первый Integration Gate.
2. **ADR-013** (Style Intelligence / `MY_STYLE_ID`) — явно помечено как блокирующее в ACS-009, требуется до Gate 4.
3. **ADR-008** и **ADR-009** (порядок Domain Intelligence и разрешение конфликта Layout↔Furniture) — оба также перед Gate 4, логически связаны с ADR-013 (все три определяют, как работает цепочка Domain Intelligence).
4. **ADR-012** (граница Material/Color) — перед реализацией соответствующих модулей, не привязано к конкретному Gate жёстко.
5. **ADR-007** (параллелизм помещений) — перед Gate 6.
6. **ADR-011** и **ADR-014** (связь с Quality Intelligence, лимит Refinement) — перед Gate 9.
7. **ADR-005** (Design Reasoning) — не блокирует ни один Gate напрямую, но логически зависит от готовности ADR-010 (decisionTrace) и решений по Domain Intelligence — реализуется позже остальных.
8. **Часть 2** (сверка с ADR_INDEX.md) — может выполняться параллельно с любым из пунктов выше, так как не имеет содержательной зависимости.
